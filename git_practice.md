# 📂 Git 강의장-집 가이드

---

## 1️⃣ 로컬 저장소와 원격 저장소 연결하기

상황에 따라 아래 두 가지 방법 중 하나를 선택하여 진행합니다.

### ✅ 방법 A: 원격 저장소에서 시작 (Clone)

가장 권장하는 방식입니다. 이미 생성된 원격 저장소를 내 컴퓨터로 그대로 가져옵니다.

1. **원격 저장소 생성:** GitHub, GitLab 등에서 프로젝트를 먼저 생성합니다.

2. **Clone 실행:** 터미널에서 아래 명령어를 실행합니다.
   
   ```bash
    git clone {remote_url} .
    ex) git clone https://lab.ssafy.com/s15/d02/gumi02.git
   ```

---

### ✅ 방법 B: 로컬 저장소에서 시작 (Remote Add)

이미 내 컴퓨터에 작업 중인 폴더가 있을 때 사용합니다.

1. **로컬 설정:** 작업 폴더에서 `git init`을 실행하여 로컬 저장소를 만듭니다.

2. **원격 생성:** **비어있는** 원격 저장소를 생성합니다. (README.md 등을 만들지 마세요.)

3. **저장소 연결:** 로컬과 원격 저장소를 연결합니다.
   
   ```bash
   git remote add origin {remote_url}
   # 예시: git remote add origin https://lab.ssafy.com/s15/d02/gumi02.git
   ```

4. **첫 Push:** 로컬의 내용을 원격에 반영합니다.
   
   ```bash
   git push -u origin master  # 환경에 따라 master 대신 main 사용
   ```

# 📂 Git 동기화 가이드 (시나리오)

집과 강의장을 오가며 코드를 관리할 때 발생할 수 있는 정상적인 흐름과 충돌 상황에 대한 정리입니다.

---

## 🛠 사전 준비 사항

1. **원격 저장소(Remote Repo):** GitHub, GitLab 등 생성
2. **로컬 저장소 1(Local 1):** 집(Home) PC
3. **로컬 저장소 2(Local 2):** 강의장(Class) PC
   - *Tip: 원격 저장소를 먼저 만든 후 각 로컬에서 `git clone`하는 것이 가장 편리합니다.*

---

## ✅ 1. 정상 동작 흐름

가장 이상적인 순환 구조입니다: **"작업 후 Push, 시작 전 Pull"**

1. **로컬 1 (집):** 코드 수정 후 저장
   - `git add .`
   - `git commit -m "집에서 작업 완료"`
2. **로컬 1 (집) → 원격:** 서버로 업로드
   - `git push origin main`
3. **로컬 2 (강의장) ← 원격:** 서버에서 내려받기
   - `git pull origin main`
4. **로컬 2 (강의장):** 이어서 작업 후 다시 **Push**

> **결과:** 집 ↔ 원격 ↔ 강의장 순서가 꼬이지 않아 데이터가 안전하게 동기화됩니다.

---

## ⚠️ 2. 비정상 동작 및 충돌 (Conflict Scenario)

Push나 Pull을 잊었을 때 발생하는 전형적인 에러 상황입니다.

### 발생 과정

1. **로컬 1 (집):** `Commit 1` 생성 후 `push` 완료.
2. **로컬 2 (강의장):** `pull` 수행 (현재 `Commit 1` 상태) — **여기까지 정상**
3. **로컬 2 (강의장):** `Commit 2` 생성 (앗! 작업 후 **push를 깜박함!** 😲)
4. **로컬 1 (집):** 새로운 작업 수행 후 `Commit 2-1` 생성.
5. **로컬 1 (집) → 원격:** `push` 성공 (원격은 이제 `Commit 2-1` 상태).
6. **로컬 2 (강의장):** 뒤늦게 원격의 내용을 가져오려고 `pull` 시도.

### 🔴 에러 발생

* **원인:** 로컬 2의 역사(Commit 2)와 원격 저장소의 역사(Commit 2-1)가 서로 갈라졌기 때문입니다.
* **메시지:** `Rejected`, `Merge Conflict` 또는 `fetch first` 관련 에러가 발생합니다.

---

## 💡 해결 방법 및 예방 수칙

### 🛠 충돌 해결하기

이미 에러가 발생했다면, Git의 안내에 따라 **Merge(병합)** 과정을 거쳐야 합니다.

1. `git pull` 시 발생하는 충돌 파일 수정.
2. 다시 `git add`, `git commit`.
3. 마지막으로 `git push`.

### 🚀 예방

1. **출근/등교 후:** 자리에 앉자마자 `git pull`부터 실행하세요.
2. **퇴근/하교 전:** `git push`를 했는지 반드시 확인하세요.
3. **상태 확인:** 내가 지금 어디 있는지 헷갈린다면 늘 `git status`를 확인하는 습관을 들입시다.
