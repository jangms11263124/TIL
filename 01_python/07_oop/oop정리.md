# 객체 지향 프로그래밍(OOP)

---

## 1. 객체 지향 프로그래밍 (OOP) 기초

**객체 지향 프로그래밍**은 프로그램을 데이터(변수)와 그 데이터를 처리하는 함수(메서드)를 하나의 단위(객체)로 묶어 관리하는 방식

* **클래스 (Class)**: 객체를 만들기 위한 **설계도**이며, 새로운 타입을 만드는 수단
* **인스턴스 (Instance)**: 클래스를 통해 생성된 **실제 객체**
* **이름 공간 (Namespace)**: 클래스와 인스턴스는 각각 독립적인 이름 공간을 가지며, 속성 접근 시 **인스턴스 → 클래스** 순으로 탐색

---

## 2. 메서드 (Method) 활용

메서드는 클래스 내부에 정의된 함수로, 객체의 동작을 코드로 표현한 것

| 종류           | 첫 번째 인자 | 주요 특징                                        |
|:------------ |:------- |:-------------------------------------------- |
| **인스턴스 메서드** | `self`  | 인스턴스의 상태를 조작하거나 동작을 수행함.                     |
| **클래스 메서드**  | `cls`   | `@classmethod` 사용. 클래스 변수 조작 및 클래스 레벨 동작 수행. |
| **스태틱 메서드**  | 없음      | `@staticmethod` 사용. 클래스/인스턴스와 무관하게 독립적으로 동작. |

> **생성자 메서드 (`__init__`)**: 인스턴스 생성 시 자동 호출되어 인스턴스 변수의 초기값을 설정합니다.

---

## 3. 클래스 상속 (Inheritance)

기존 클래스(부모)의 속성과 메서드를 새로운 클래스(자식)가 물려받아 코드 재사용성을 높이는 기법

* **메서드 오버라이딩 (Overriding)**: 부모의 메서드를 자식 클래스에서 동일한 이름으로 **재정의**하여 동작을 바꾸는 것
* **다중 상속 & MRO**: 둘 이상의 클래스를 상속받을 때, **MRO(메서드 결정 순서)** 알고리즘에 따라 탐색 순서를 결정
* **`super()` 메서드**: MRO에 따라 부모 클래스의 메서드나 속성에 접근할 수 있게 해주는 내장 함수
